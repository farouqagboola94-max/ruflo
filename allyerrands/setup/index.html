Step 1 — Supabase

Your database, realtime engine, and Edge Functions all live here.

Create Project

  1. Go to supabase.com → New Project

  2. Name it za-allyerrands. Choose a strong DB password.

  3. Region: West EU (Ireland) or US East — closest to Nigeria.

  4. Wait ~2 minutes for it to spin up.

Get Your Credentials

  1. Settings → General → copy Reference ID

  2. Settings → API → copy anon public key

  3. Same page → copy service_role key (keep secret!)

Enter Credentials

Settings → General → Reference ID

Settings → API → anon public

Settings → API → service_role (never share publicly)

PIN to access the admin dashboard at /admin

Step 2 — Flutterwave

Handles payment links and webhook confirmation when clients pay.

Get API Keys

  1. Go to dashboard.flutterwave.com → log in

  2. Settings → API Keys → copy your Secret Key

  3. Start with Test mode — key starts with FLWSECK_TEST-

Set Webhook (after Step 5)
⚡ Set this after running deploy.sh — you'll have the URL then.

  1. Settings → Webhooks in Flutterwave

  2. URL: https://YOUR_PROJECT_ID.supabase.co/functions/v1/flw-webhook

  3. Secret Hash = whatever you set below as FLW Webhook Secret

Enter Credentials

Make this up — use the same value in Flutterwave's webhook settings

Step 3 — WhatsApp Business API

Sends automated messages to clients and Phalanx at every status change.

Option A — Meta (Official)

  1. Go to developers.facebook.com → My Apps → Create App → Business

  2. Add WhatsApp product to the app

  3. WhatsApp → API Setup: copy Temporary Access Token + Phone Number ID

  4. For production: Business Settings → System Users → generate permanent token with whatsapp_business_messaging

Option B — WHAPI.cloud (Easier)

  1. Go to whapi.cloud → sign up → create a channel → scan QR

  2. Get your API Token and Channel ID

  3. Note: using WHAPI needs a small Edge Function update — message me and I'll swap it.

Enter Credentials

Numeric Phone Number ID from Meta's WhatsApp API Setup page

Step 4 — Make.com Scenarios

Two automated scenarios alert your Phalanx dispatcher via WhatsApp.

Scenario 1 — New Errand Request

  1. Go to make.com → Create a new Scenario

  2. Add: Webhooks → Custom Webhook → Create a webhook → copy URL

  3. Add: WhatsApp Business Cloud → Send a Text Message
    Message: 🚨 NEW ERRAND — {{1.client_name}} needs {{1.item_description}} from {{1.pickup_address}}. Payment link sent. Standby.

  4. Turn the scenario ON → paste URL below

Scenario 2 — Payment Confirmed

  1. Create another scenario → Webhooks → Custom Webhook → copy URL

  2. Add: WhatsApp Business Cloud → Send a Text Message
    Message: ✅ PAYMENT CONFIRMED — DISPATCH NOW\n{{1.client_name}} | {{1.pickup_address}} → {{1.dropoff_address}}

  3. Turn ON → paste URL below

Step 5 — Deploy

Your filled deploy.sh is ready. Run it once on your machine.

Run It

  1. Install Supabase CLI: brew install supabase/tap/supabase (Mac) or see supabase.com/docs/guides/cli

  2. Save the script above as deploy.sh in your allyerrands/ folder

  3. Run:

    cd allyerrands\nbash deploy.sh

  4. Copy the printed flw-webhook URL → paste into Flutterwave → Settings → Webhooks
