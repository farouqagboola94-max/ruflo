Za.allyErrands Dispatch API

The Za.allyErrands Dispatch API lets your business programmatically create and track same-day deliveries across Lagos. Integrate it into your e-commerce checkout, ERP, or ops dashboard — and our runner network handles the rest.

All requests use HTTPS. Responses are JSON. Authentication uses Bearer tokens issued per business account.

Base URL: All API requests should be made to https://api.zallyerrands.com/v1
Beta notice: The API is currently in closed beta. Sign up for the Business Plan to receive API access.
// AUTHENTICATION

Authentication

The API uses Bearer token authentication. Include your API key in the Authorization header of every request.

Header 
Authorization: Bearer za_live_sk_xxxxxxxxxxxxxxxxxxxx

Keys are prefixed with za_live_sk_ for production and za_test_sk_ for sandbox. Test keys never trigger real dispatches or charges.

Tip: Rotate your API keys anytime from the Business Portal → API Keys tab.

Create a Dispatch

Creates a new dispatch request. A runner will be assigned and notified within seconds of receiving the request during operating hours.

POST /v1/dispatch Creates a new dispatch

Request Body

Parameter Type Required Description
pickup_address string Required Full street address for pickup. Include LGA or landmark for accuracy.
dropoff_address string Required Full street address for delivery.
item_description string Required What is being dispatched. Max 500 chars.
recipient_name string Required Name of the person receiving the delivery.
recipient_phone string Required Recipient's phone number (E.164 format, e.g. +2348012345678).
priority string Optional standard (default) or express. Express guarantees dispatch within 10 minutes.
scheduled_at string Optional ISO 8601 timestamp. Omit for immediate dispatch.
reference string Optional Your internal order ID or reference. Returned on all events for reconciliation.
notes string Optional Instructions for the runner. Max 200 chars.
cURL 
curl -X POST https://api.zallyerrands.com/v1/dispatch \
  -H "Authorization: Bearer za_live_sk_xxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "pickup_address": "14 Adeola Odeku Street, Victoria Island, Lagos",

    "item_description": "3 bags of groceries from Shoprite",

    "priority": "standard",
    "reference": "ORD-20260524-001"

Get a Dispatch

Retrieves a single dispatch by its ID. Includes the assigned runner, current status, and live ETA.

GET /v1/dispatch/:id Returns a single dispatch

Path Parameters

ParameterTypeRequiredDescription
id string Required Dispatch ID returned from POST /v1/dispatch.
cURL 
curl https://api.zallyerrands.com/v1/dispatch/dsp_01HX3KPQRS \
  -H "Authorization: Bearer za_live_sk_xxxx"
const dispatch = await client.dispatch.get('dsp_01HX3KPQRS');
console.log(dispatch.status); // "en_route"

List Dispatches

Returns a paginated list of all dispatches for your account, ordered by creation time (newest first).

GET /v1/dispatches Paginated list of dispatches

Query Parameters

ParameterTypeRequiredDescription
limit integer Optional Number of results per page. Default 20, max 100.
cursor string Optional Cursor for pagination. Use next_cursor from the previous response.
status string Optional Filter by status: queued, dispatched, en_route, delivered, cancelled, failed.
from string Optional ISO 8601 date. Return dispatches created after this date.
to string Optional ISO 8601 date. Return dispatches created before this date.
cURL 
curl "https://api.zallyerrands.com/v1/dispatches?status=en_route&limit=10" \
  -H "Authorization: Bearer za_live_sk_xxxx"

Response

JSON 
{

  ],
  "count": 2,
  "next_cursor": null,
  "has_more": false
}

Cancel a Dispatch

Cancels a dispatch that hasn't yet been picked up. Once a runner has confirmed pickup, cancellation is not possible via the API — contact support.

DEL /v1/dispatch/:id Cancel a queued or dispatched run
Note: Cancellations on dispatches with status en_route or delivered will return a 409 Conflict error.
cURL 
curl -X DELETE https://api.zallyerrands.com/v1/dispatch/dsp_01HX3KPQRS \
  -H "Authorization: Bearer za_live_sk_xxxx"
200 OKReturns the cancelled dispatch object with status: "cancelled"

Webhook Events

Za.allyErrands sends POST requests to your registered webhook URL as a dispatch moves through its lifecycle. Register your URL in the Business Portal → API Keys tab.

dispatch.queued
Fired immediately after a dispatch is created and enters the queue.
dispatch.runner_assigned
A runner has accepted the job. Includes runner name and contact details.
dispatch.picked_up
Runner has collected the item. Photo proof attached as a signed URL.
dispatch.en_route
Dispatch is in transit. ETA is updated every 5 minutes.
dispatch.delivered
Delivery confirmed. Includes delivery timestamp and photo proof URL.
dispatch.failed
Delivery failed after 3 attempts. Includes failure reason and next steps.
dispatch.cancelled
Dispatch was cancelled. Triggered whether cancelled via API or by support.

  "id":          "evt_01HXAB12CD",

Verifying Payloads

Each webhook request includes a Za-Signature header — an HMAC-SHA256 hex digest of the raw request body, signed with your webhook secret. Always verify this before processing.

Node.js 
import crypto from 'node:crypto';

function verifyWebhook(rawBody, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(rawBody)
    .digest('hex');
  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signature)
  );
}
import hmac, hashlib

def verify_webhook(raw_body: bytes, signature: str, secret: str) -> bool:
  expected = hmac.new(
    secret.encode(), raw_body, hashlib.sha256
  ).hexdigest()
  return hmac.compare_digest(expected, signature)

Node.js SDK

The official Node.js SDK wraps all API endpoints and webhook verification with TypeScript types.

Install 
npm install @zallyerrands/sdk
TypeScript 
import AllyErrands, { Dispatch } from '@zallyerrands/sdk';

const client = new AllyErrands({
  apiKey: process.env.ZALLYERRANDS_API_KEY!
});

// Typed dispatch creation
const dispatch: Dispatch = await client.dispatch.create({
  pickupAddress:    '14 Adeola Odeku, VI',
  dropoffAddress:   '42 Admiralty Way, Lekki',
  itemDescription: 'Documents',
  recipientName:   'Amara Okafor',
  recipientPhone:  '+2348012345678'
});

// Iterate all active dispatches
for await (const d of client.dispatch.list({ status: 'en_route' })) {
  console.log(d.id, d.eta_minutes);
}

Rate Limits

Rate limits are enforced per API key per rolling 60-second window. Exceeding a limit returns 429 Too Many Requests. Limits reset automatically.

Business
120 req/min
Enterprise
600 req/min
Sandbox
30 req/min

Rate limit headers are included on every response:

Response Headers 
X-RateLimit-Limit:     120
X-RateLimit-Remaining: 87
X-RateLimit-Reset:     1716545460

Error Codes

All errors return a JSON body with a machine-readable code and a human-readable message.

Error shape 
{
  "error": {
    "code":    "DISPATCH_NOT_CANCELLABLE",
    "message": "Cannot cancel a dispatch with status en_route.",
    "status":  409
